Wie ich meine professionelle CV-Website mit AWS, Terraform & GitHub Actions erstellt und bereitgestellt habe

Autor: Ibrahim Kilicaslan | Datum: 22.07.2025

Als Cloud & DevOps Engineer wollte ich, dass meine persönliche Website nicht nur meine berufliche Erfahrung, sondern auch meine technischen Fähigkeiten in Automatisierung, Cloud-Infrastruktur und bewährten Praktiken widerspiegelt. Anstatt einen einfachen Website-Builder zu verwenden, entschied ich mich, meine CV-Website mit AWS-Services (S3, Route 53, CloudFront, ACM) zu erstellen und alles mit Terraform und GitHub Actions zu automatisieren.

In diesem Beitrag teile ich meine Reise, die Architektur, die Herausforderungen, denen ich mich gestellt habe, und eine Schritt-für-Schritt-Anleitung für alle, die eine moderne, sichere und vollständig automatisierte statische Website auf AWS erstellen möchten.

Warum nicht einfach einen Website-Builder verwenden?

  • Vollständige Kontrolle über Infrastruktur und Bereitstellung
  • Lernen und Präsentieren realer DevOps- und Cloud-Fähigkeiten
  • Skalierbarkeit und Sicherheit mit AWS-Best-Practices
  • Automatisierung: Keine manuellen Uploads, kein FTP, keine "Klick-und-Hoffen"-Bereitstellungen

Architektur-Übersicht

Die Website ist eine statische HTML/CSS/JS-Site, gehostet auf AWS S3, global über CloudFront ausgeliefert, mit HTTPS (ACM) gesichert und DNS mit Route 53 verwaltet. Die gesamte Infrastruktur ist mit Terraform als Code definiert, und Bereitstellungen werden mit GitHub Actions automatisiert.

  • S3: Statisches Website-Hosting
  • CloudFront: CDN, HTTPS, benutzerdefinierte Domains
  • Route 53: DNS-Verwaltung
  • ACM: Kostenlose SSL-Zertifikate (automatische Verlängerung)
  • GitHub Actions: CI/CD für automatische Bereitstellung

Schritt für Schritt: Wie ich es gemacht habe

  1. Design & Frontend: Erstellung einer modernen, responsiven CV-Website mit HTML, CSS und JavaScript.
  2. Infrastructure as Code: Erstellung eines modularen Terraform-Setups für S3, CloudFront, Route 53 und ACM.
  3. CI/CD: Einrichtung von GitHub Actions für automatisierte Bereitstellungen.
  4. Domain & DNS: Registrierung und Verwaltung meiner Domain mit Route 53.
  5. SSL & Sicherheit: Verwendung von ACM für SSL-Zertifikate und Erzwingung von HTTPS.

Abschließende Gedanken

Der Aufbau einer eigenen Website mit AWS und vollständiger Automatisierung ist nicht nur eine großartige Lernerfahrung – es ist auch eine Möglichkeit, sich als DevOps- oder Cloud-Engineer hervorzuheben.

Wenn Sie eine moderne, sichere und vollständig automatisierte persönliche Website möchten, nutzen Sie gerne mein Repository als Vorlage und machen Sie es zu Ihrem eigenen!
